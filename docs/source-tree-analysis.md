## Critical Directories

| Directory | Purpose | Refactor Sensitivity |
| --- | --- | --- |
| `module/` | Runtime behavior loaded by Foundry | High. Depends on Foundry globals and lifecycle |
| `module/actor/` | Actor data preparation and actor sheet UI | High. Central to character state and interaction |
| `module/item/` | Item preparation, item sheets, combat/vehicle/cyberware behavior | High. Current largest behavior concentration |
| `templates/` | Sheet, dialog, and chat rendering | High. Paths are hardcoded in JS/helpers/preload list |
| `scss/` | Source styling | Medium. Must compile to tracked CSS |
| `packs/` | Default skills, weapons, armor, cyberware, gear, vehicles | High. Data asset changes can affect existing worlds and installs |
| `lang/` | Localization keys | Medium. Missing keys can surface directly in UI |

## Entry Points

- `system.json` is the Foundry package entry contract.
- `module/cyberpunk2020.js` is the runtime entry module.
- `scss/cyberpunk2020.scss` is the styling source entry.
- `template.json` defines actor/item defaults and schema-like data structure.

## Path Coupling

Several paths are hardcoded or generated by helpers:

- Sheet templates: `systems/cyberpunk2020/templates/...`
- Chat templates: `systems/cyberpunk2020/templates/chat/...`
- Dynamic item partials: `templates/item/parts/[item.type]/summary.hbs` and `settings.hbs`
- Field partials: `templates/fields/string.hbs`, `number.hbs`, `boolean.hbs`, `select.hbs`
- Compendium lookup: `game.packs.get("cyberpunk2020.default-skills")`

Path changes should be treated as package-contract changes, not cosmetic moves.
